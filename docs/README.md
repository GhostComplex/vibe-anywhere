# docs/

Project documentation for Vibe Anywhere.

## Structure

```
docs/
├── README.md          # This file — documentation guide
└── prd/               # Product Requirement Documents
    └── PRD-*.md       # One PRD per feature/milestone
```

## PRD Lifecycle

1. **Draft** — Initial design, open for discussion
2. **In Review** — Shared with stakeholders for feedback
3. **Approved** — Ready for implementation
4. **Implemented** — Code shipped, doc kept as reference
5. **Superseded** — Replaced by a newer design

## Conventions

- One PRD per milestone or feature
- Filename: `PRD-<feature-name>.md`
- All content in English
- Update PRD status as it progresses through the lifecycle
